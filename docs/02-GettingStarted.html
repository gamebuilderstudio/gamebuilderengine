Getting Started

PushButton Engine is designed to be very easy to drop and start using. This chapter explains how you can start using it immediately with a variety of Flash development tools.

The rest of the chapters in this manual explain how to use PBE.

Precompiled SWC?

If you already have a Flash development environment set up and want to use PBEngine as a library, all you need to do is grab the precompiled PBEngine.swc from the bin/ folder. The swc is only included in the zip'ed downloads from our download page, and is not checked in to SVN. Go to the download page to get a zip with a precompiled PBEngine.swc.

If you don't have a Flash development environment setup, keep reading.

ActionScript or Flex/MXML?

The Engine will work in projects that only use ActionScript as well as projects that use the Flex SDK. You are free to do whatever best fits the needs of your project. If this doesn't mean anything to you, skip to the next section.

We include two basic demos/game templates. PBFlexlessEngineDemo is an example of how you will want to set up your ActionScript files for a basic game that does not use Flex. PBEngineDemo is the same, but using Flex. Both demos are included in the SDK download, in the examples/ folder.

Building with FlashDevelop

FlashDevelop is a free Windows-based IDE for developing in Flash. To start a PBEngine project with it:

  1. Launch FlashDevelop.
  2. Choose Project -> New Project...
  3. Choose the "AS3 Project" template. Fill in the name/path textboxes with appropriate values (like TestProject or the name of your game).
  4. Copy bin/PBEngine.swc into the project by drag-and-dropping it onto the lib/ folder in FlashDevelop's Project pane. Right click on it and choose "Add To Library." The item will turn a bright blue, so that you know that PushButton Engine is available for your use.

If you'd like to run a simple example game, paste the following code into your Main.as file in the project pane's src/ folder:

package 
{
	import flash.display.Sprite;
	import com.pblabs.engine.PBE;
	import com.pblabs.engine.debug.*;
	
	public class Main extends Sprite 
	{
		public function Main():void 
		{
			PBE.startup(this);
			Logger.print(this, "Hello, world!");
		}
	}
}

If you run this, you'll see the "Hello, world!" message show up in the log pane at the bottom of FlashDevelop. You can also hit tilde to view the PBE console, which will show Hello, world! along with other information.

Using the Flex SDK

Once you have installed the free, cross-platform Flex SDK, it's easy to use PBEngine. Just copy the contents of the src/ folder to whever you are building your MXML or .as files.

For instance, if you create the following HelloWorld.mxml file and copy everything from src/ into the same directory:

<?xml version="1.0" encoding="utf-8"?>
<mx:Application xmlns:mx="http://www.adobe.com/2006/mxml" applicationComplete="kickoff()">
   <mx:Script>
      <![CDATA[
         import com.pblabs.engine.PBE;
         import com.pblabs.engine.debug.*;
         
         public function kickoff():void
         {
            PBE.startup(this);
            Logger.print(this, "Hello, world!");
         }
      ]]>
   </mx:Script>
</mx:Application>

You can compile it with mxmlc HelloWorld.mxml to produce HelloWorld.as. If you hit tilde you will be able to see "Hello, world!" show up in the log.

You can also compile .as files with MXMLC. Consider this file, HelloWorld.as:

package
{
	import flash.display.Sprite;
	import com.pblabs.engine.PBE;
	import com.pblabs.engine.debug.*;
	
	public class HelloWorld extends Sprite 
	{
		public function Main():void 
		{
			PBE.startup(this);
			Logger.print(this, "Hello, world!");
		}
	}
}

You can compile this into HelloWorld.swf by running mxmlc HelloWorld.as.

Using Ant

The official SDK release is built using Ant. To build the SDK, have the

Apache Ant is a command line, make-like tool which along with the free Flex SDK can compile the Engine and related demos and lessons. Go to the build/ folder and type ant to build the SDK.

You will notice build.xml files in each demo and lesson folder. Running ant in these folders will compile the respective demo or lesson.

You can add and compile your own applications by copying one of the demos or lessons. To get a fresh project to modify, rename the folder (for instance from PBEngineDemo/ to MySweetGame/), then rename the root class in src/ (for instance from PBEngineDemo.mxml to MySweetGame.mxml). Finally, open the build.xml file in the folder (ie MySweetGame/build.xml) and replace every instance of the original project name with the new name (for instance PBEngineDemo to MySweetGame). Now when you run ant in the project's folder, it will compile the project for you using the Flex SDK.

Using Flex Builder

Adobe Flex Builder is an Eclipse-based IDE for ActionScript development. It's very easy to use with PBEngine.

Note: Flex Builder Plug-in doesn't work with every version of Eclipse. We recommend using the standalone version of Flex Builder for maximum compatibility.

To open a project with Flex Builder, perform the following steps:

  1. Launch Flex Builder.
  2. Right click in the project browser and choose New -> Flex Project...
  3. Enter the name of the project you are going to open (for instance, PBEngineDemo).
  4. For the project location, uncheck "use default location", then for the location enter the path to or browse to the project's folder (for instance C:\PBEngine\examples\PBEngineDemo).
  5. Click Next several times to get through the rest of the wizard.
  6. Right click on the project in the Flex Navigator pane.
  7. Choose the Flex Compiler tab in the properties window.
  8. Add --keep-as3-metadata+=TypeHint,EditorData,Embed to the end of the "Additional compiler arguments" text box.

You have to include PushButton Engine in one of two ways so you can compile the example. There are two options:

  1. If you don't intend to modify the core engine code, you can take this option. Flex Builder will create a lib/ folder in the project directory. Copy PBEngine.swc into this folder, then right click the project in Flex Builder and choose Refresh. The SWC will show up, and the project will compile succesfully.
  2. If you intend to modify/browse core engine code, take this option. Right click the project, choose Properties, then select Flex Build Path, go to the Source Path pane, and hit Add Folder. Select the root PBEngine src/ folder. Now you can modify the core engine files, and the changes will take effect when you recompile.

After doing either option 1 or 2, you should be able to compile and run your project.

SDK Layout

Now that you are able to build, let's talk about how files in the SDK are organized.

  • build/ - Contains Ant files used to build SDK releases.
  • docs/ - Contains this manual, which covers the core engine released by PushButton Labs. docs/api contains ASDoc-generated API references.
  • bin/ - PBEngine.swc is built here.
  • src/ - The source for the core PushButton Engine libraries. If you get source for additional components, drop it in here.
  • examples/ - Contains several example games.
  • lessons/ - Contains projects related to the PBE lessons.

Project Layout

Each project contained in the SDK has the same file structure. The structure is based on standard Flash project structure, so it can be easily opened in Flex Builder, Flash, or other tools. Projects compile to SWFs.

There are three main folders for each project.

  • src/ - contains the project's source code.
  • assets/ - holds the various art, sound, and level files for the project.
  • bin/ - holds the compiled SWF for the project.
